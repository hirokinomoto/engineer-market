# ITエンジニア市場データELTプロジェクト 作業メモ

## 目的

本プロジェクトは、ITエンジニア職種に関する検索関心データと求人需要データを収集し、SQLを中心に `raw → staging → mart` の流れで整形・集計するためのポートフォリオ作成用メモです。

公開用READMEではなく、作業工程・判断事項・タスクを整理するための内部メモとして使用します。

## 初期版の対象範囲

### 対象期間

- 2022年4月〜2026年3月

### 使用予定データソース

| データ種別 | データソース | 用途 |
|---|---|---|
| 求人需要データ | e-Stat / 厚生労働省 職業安定業務統計 | IT系職業の有効求人数推移を確認する |
| 検索関心データ | Google Trends | エンジニア職種別の検索関心推移を確認する |

### 初期版で扱う分析・可視化

1. e-Stat / 厚生労働省 職業安定業務統計によるIT転職市場の全体推移
   - ソフトウェア開発技術者の有効求人数
   - 情報処理・通信技術者の有効求人数
   - 通信ネットワーク技術者の有効求人数

2. 職種別の検索関心推移
   - ソフトウェアエンジニア
   - インフラエンジニア
   - ネットワークエンジニア
   - クラウドエンジニア
   - データベースエンジニア
   - データエンジニア
   - AIエンジニア

3. 「転職」KWの検索関心比較
   - ソフトウェアエンジニア 転職
   - インフラエンジニア 転職
   - ネットワークエンジニア 転職
   - クラウドエンジニア 転職
   - データベースエンジニア 転職
   - データエンジニア 転職
   - AIエンジニア 転職

## 初期版でやらないこと

- 現在求人のスキル出現率集計
- 求人票スクレイピング
- 企業別求人分析
- 職務内容テキスト分類
- 高度な統計分析
- 需要予測
- Pythonによる自動取得・自動ロード

※ Python要素は、SQL版が完成した後に追加する。

## プロジェクトの主目的

本プロジェクトの主目的は、分析結果を深掘りすることではなく、以下の工程を見せることです。

- 外部データを収集する
- rawデータとして保存する
- DBに取り込む
- SQLでstaging層に整形する
- SQLでmart層を作成する
- データ品質チェックを行う
- 可視化用データを作成する

## 注意点

- Google Trendsの値は実検索回数ではなく、検索関心指数として扱う
- e-Statの職業分類と検索KWの職種名は完全には一致しない
- 求人需要と検索関心の関係は因果ではなく、傾向比較として扱う
- 初期版では2022年4月〜2026年3月に絞り、長期化は後から行う
- 分析の深掘りより、データ取得・整形・集計工程の再現性を重視する
