# BIレポート構成メモ

## 目的

本ドキュメントは、Looker Studioで作成した可視化レポートの構成を整理した補足資料です。

本プロジェクトでは、BigQuery上に作成したmartテーブルをLooker Studioへ接続し、ITエンジニア職種に関する検索関心と求人需要の推移を可視化しました。

現在のレポートは、単にグラフを並べるのではなく、採用担当者や面接官が短時間で以下を理解できるように、提示型ダッシュボードとして構成しています。

- 何を目的に分析したか
- どのデータを使ったか
- どのようにELTで整備したか
- どのような傾向が見えるか
- このデータでは何が判断できないか
- 可視化・実装で何を意識したか

## レポートの位置づけ

本プロジェクトでは、成果物の中心をLooker Studioレポートに置いています。

| 位置づけ | 内容 |
|---|---|
| Looker Studio | 主成果物。分析の問い、可視化結果、データの限界を提示する |
| GitHub README | プロジェクト全体の入口。成果物と詳細ドキュメントへの導線を示す |

Looker Studioは、操作して深掘りする探索型ダッシュボードというより、上から順に読むことで全体像を理解できる提示型ダッシュボードとして設計しています。

## 使用データ

Looker Studioでは、BigQueryの以下テーブルを使用しました。

```text
engineer-market.engineer_market.mart_engineer_market_yearly
```

このテーブルは、Google Trendsとe-Statのデータを年次粒度で統合したmartテーブルです。

### 主なフィールド

| フィールド | 内容 |
|---|---|
| year | 年 |
| data_source | データソース |
| category_type | カテゴリ種別 |
| category | 職種名、未経験IT職関連KW、または職業分類名 |
| metric_name | 指標名 |
| value | 指標値 |
| aggregation_method | 年次集計方法 |
| unit | 単位 |

### category_type

| category_type | 内容 |
|---|---|
| role_name | 職種名KW |
| beginner_query | 未経験IT職関連KW |
| occupation | e-Stat職業分類 |

---

## 対象期間

```text
2013年1月〜2025年12月
```

Looker Studioでは、月次データを年平均に集計した `mart_engineer_market_yearly` を使用しています。

年次martの行数は以下です。

| data_source | category_type | 行数 | カテゴリ数 | 指標数 |
|---|---|---:|---:|---:|
| estat | occupation | 39 | 1 | 3 |
| google_trends | beginner_query | 39 | 3 | 1 |
| google_trends | role_name | 91 | 7 | 1 |

合計行数は169行です。

# 3. データとELT構成

## 役割

使用データと、raw → staging → adjusted → mart → Looker の処理フローを示すページです。

## 使用データ

| データ | 内容 |
|---|---|
| Google Trends | 職種名KW・未経験IT職関連KWの検索関心指数 |
| e-Stat 職業安定業務統計 | 情報処理・通信技術者の新規求人・有効求人・有効求人倍率 |

## 処理フロー

```text
raw
  ↓
staging
  ↓
adjusted
  ↓
mart
  ↓
Looker
```

## 意図

データを単に可視化しただけでなく、外部データをBigQueryで扱いやすい形に整備し、BI用のmartまで作成していることを示しています。

また、Google Trendsは取得グループごとに0〜100で正規化されるため、共通KWである `システムエンジニア` をアンカーとして補正したことも明示しています。

# 4. IT関連職種の求人需要 推移

## 役割

企業側の求人需要を確認するページです。

## 使用データ

| 項目 | 内容 |
|---|---|
| データソース | e-Stat 職業安定業務統計 |
| 対象職業分類 | 情報処理・通信技術者 |
| 対象指標 | 新規求人、有効求人、有効求人倍率 |
| 対象期間 | 2013〜2025年 |

## グラフ

### 新規求人・有効求人

| 項目 | 設定 |
|---|---|
| グラフ種類 | 折れ線グラフ |
| ディメンション | year |
| 内訳ディメンション | metric_name |
| 指標 | value |
| 条件 | data_source = estat / metric_name = 新規求人 または 有効求人 |

### 有効求人倍率

| 項目 | 設定 |
|---|---|
| グラフ種類 | 折れ線グラフ |
| ディメンション | year |
| 内訳ディメンション | なし、または metric_name |
| 指標 | value |
| 条件 | data_source = estat / metric_name = 有効求人倍率 |

## 注記

```text
この分類はデータエンジニア単体の求人件数ではなく、IT関連職種全体の求人需要を見る補助指標です。
```

## 意図

e-Statの職業分類は、データエンジニア単体ではなく「情報処理・通信技術者」です。  
そのため、IT関連職種全体の求人需要を見る補助指標として扱っています。

単位が異なるため、求人件数と有効求人倍率は別グラフに分けています。

# 7. 公開データから分かること・分からないこと

## 役割

分析結果と、公開データだけでは判断できない内容を整理するページです。

## 分かること

- IT関連職種全体の求人需要の推移
- 職種名への検索関心の推移
- 未経験IT職関連KWの検索関心の推移

## 判断できないこと

- データエンジニア単体の求人件数
- 未経験歓迎求人の実数
- 年収レンジ
- 必須スキル
- 勤務地の傾向

## 次に必要なデータ

- 求人票単位のデータ
- 年収
- 雇用形態
- 勤務地
- 未経験歓迎有無
- 要求スキルの頻度

## 結論

```text
公開統計と検索関心データで市場全体の動きは確認できるが、転職難易度や参入しやすさを判断するには求人票データの追加分析が必要。
```

## 意図

このページでは、分析結果を断定しすぎず、データから判断できることと判断できないことを分けています。

特に、今回のデータだけでは「未経験からデータエンジニアに転職しやすいか」は判断できません。  
そのため、次の分析では求人票単位のデータが必要であることを示しています。

## フィルタ設計

現在のレポートは、閲覧者が操作しなくても内容を理解できる提示型ダッシュボードとして設計しています。

そのため、過度なフィルタ操作に依存しない構成にしています。

ただし、必要に応じて以下のフィールドを使うことで、グラフ単位の絞り込みができます。

| フィールド | 用途 |
|---|---|
| data_source | Google Trends / e-Statを切り替える |
| category_type | 職種名KW / 未経験IT職関連KW / 職業分類を切り替える |
| category | 表示する職種・キーワード・職業分類を絞り込む |
| metric_name | 表示する指標を絞り込む |
