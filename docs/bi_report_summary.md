# BIレポート構成メモ

## 目的

本ドキュメントは、Looker Studioで作成した可視化レポートの構成を整理した補足資料です。

本プロジェクトでは、BigQuery上に作成したmartテーブルをLooker Studioへ接続し、ITエンジニア職種に関する検索関心と求人需要の推移を可視化しました。

分析結果の深掘りよりも、BigQueryで作成したmartをBIツールに接続し、可視化に利用できる状態まで整備することを重視しました。

主に以下を確認できる構成にしています。

- BigQueryで作成したmartをBIツールへ接続できること
- 年次推移をグラフで確認できること
- 検索関心データと求人需要データを分けて可視化できること
- データソース・指標・カテゴリをフィルタで確認できること

---

## 使用データ

Looker Studioでは、BigQueryの以下テーブルを使用しました。

```text
engineer-market.engineer_market.mart_engineer_market_yearly
```

このテーブルは、Google Trendsとe-Statのデータを年次粒度で統合したmartテーブルです。

主なフィールドは以下です。

| フィールド | 内容 |
|---|---|
| year | 年 |
| data_source | データソース |
| category | 職種名または職業分類名 |
| metric_name | 指標名 |
| value | 指標値 |
| aggregation_method | 年次集計方法 |
| unit | 単位 |

## レポート構成

本レポートでは、以下の3つの折れ線グラフを作成しました。

---

## 1. 職種別の検索関心指数 推移

### 目的

Google Trendsのデータを用いて、ITエンジニア職種ごとの検索関心の年次推移を確認するためのグラフです。

### グラフ設定

| 項目 | 設定 |
|---|---|
| グラフ種類 | 折れ線グラフ |
| ディメンション | year |
| 内訳ディメンション | category |
| 指標 | value |
| 並べ替え | year 昇順 |
| データソース条件 | data_source = google_trends |
| 指標条件 | metric_name = 検索関心指数 |

### 表示対象カテゴリ

| カテゴリ |
|---|
| システムエンジニア |
| プログラマー |
| ITエンジニア |
| Webエンジニア |
| インフラエンジニア |
| データエンジニア |
| AIエンジニア |

---

## 2. 情報処理・通信技術者の求人件数 推移

### 目的

e-Statの職業安定業務統計を用いて、情報処理・通信技術者に関する求人件数の年次推移を確認するためのグラフです。

### グラフ設定

| 項目 | 設定 |
|---|---|
| グラフ種類 | 折れ線グラフ |
| ディメンション | year |
| 内訳ディメンション | metric_name |
| 指標 | value |
| 並べ替え | year 昇順 |
| サブの並べ替え | 未設定、または year 昇順 |
| データソース条件 | data_source = estat |
| 指標条件 | metric_name = 新規求人 または 有効求人 |

### 表示対象指標

| 指標 | 単位 |
|---|---|
| 新規求人 | 人 |
| 有効求人 | 人 |

### 補足

`新規求人`・`有効求人` は単位が「人」であるため、同じグラフで表示しました。

一方、`有効求人倍率` は単位が「倍」であり、同じY軸に載せると見づらくなるため、別グラフに分けました。

---

## 3. 情報処理・通信技術者の有効求人倍率 推移

### 目的

情報処理・通信技術者の有効求人倍率の年次推移を確認するためのグラフです。

### グラフ設定

| 項目 | 設定 |
|---|---|
| グラフ種類 | 折れ線グラフ |
| ディメンション | year |
| 内訳ディメンション | なし、または metric_name |
| 指標 | value |
| 並べ替え | year 昇順 |
| データソース条件 | data_source = estat |
| 指標条件 | metric_name = 有効求人倍率 |

### 表示対象指標

| 指標 | 単位 |
|---|---|
| 有効求人倍率 | 倍 |

---

## フィルタ

レポート上部に以下のフィルタを配置しました。

| フィルタ | 用途 |
|---|---|
| category | 表示する職種・職業分類を絞り込む |
| metric_name | 表示する指標を絞り込む |

### 注意点

`category` や `metric_name` のフィルタは、複数グラフに影響する可能性があります。

たとえば、`metric_name = 有効求人倍率` を選択すると、Google Trends側のグラフは `検索関心指数` が除外されるため表示されない可能性があります。

必要に応じて、フィルタの適用範囲をグラフ単位で調整します。

---

## レイアウト

本レポートでは、以下の順番でグラフを配置しました。

```text
上段：職種別の検索関心指数 推移
中段：情報処理・通信技術者の求人件数 推移
下段：情報処理・通信技術者の有効求人倍率 推移
```

この順番にした理由は、検索関心と求人需要を分けて確認しやすくするためです。

```text
検索関心
↓
求人件数
↓
求人倍率
```

現時点で、以下を確認できる状態にしています。

- BigQueryの年次martをLooker Studioへ接続できている

- e-Statの有効求人倍率を別グラフで可視化できている
